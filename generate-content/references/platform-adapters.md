# Platform Adapters - Content Pipeline

Rules for adapting a blog post into platform-specific posts. Read alongside your project's platform style guides.

---

## Blog -> Telegram

### Transformation

1. Shorten to 1000-2000 characters
2. Punchier and more informal than the blog
3. Emoji: 1 in title, 1-2 in body (maximum 3)
4. Short paragraphs: 2-4 lines, blank line between them
5. Remove detailed explanations - keep the essence
6. Code blocks only if critical for understanding

### Structure

```
**[Emoji] [Title with result/number]**

[Context: 1-2 sentences]

**What I did:**
[Steps briefly]

**Result:**
[Outcome + insight]

-> Full post: [blog link]
```

### CTA

- Link to the full blog post
- Soft, not salesy
- Example: "More details with prompts and configs - on the blog"
- **URL format:** adapt to your blog URL structure (e.g., `https://yourdomain.com/blog/YYYY-MM-DD-slug`)

### Formatting

- `**bold**` for titles and key phrases
- Bullets via emoji or bullet character
- Short dash (-), not em-dash
- Markdown links: `[text](URL)`

## Blog -> LinkedIn

### Transformation

1. 1000-2000 characters
2. Professional framing, no slang
3. Arrows -> instead of bullets
4. Separators (blank line) between blocks
5. Add professional context (business value, industry angle)
6. Practical conclusion or takeaway at the end (no cliche engagement questions like "Have you tried this?")

### Structure

```
[HOOK - first 2 lines visible before "...more", must grab attention]

[Context with professional accent]

What I did:

-> [Step/result 1]
-> [Step/result 2]
-> [Step/result 3]

Result: [measurable outcome]

Takeaway for [your audience]:
[Practical insight]

#Hashtag1 #Hashtag2 #Hashtag3
```

### Hashtags

3-5 hashtags, generated **based on post content**:

**Selection algorithm:**
1. Extract key technologies/tools from text
2. Add 1-2 broader tags for reach
3. Prefer specific tags over generic

### Emoji

0-1 per post. Can be fully without.

---

## Common Rules for All Platforms

- Don't retell the blog verbatim - adapt
- Keep the key insight and numbers
- Remove the Sources section (it's in the blog)
- Short dash (-) everywhere
- First person
