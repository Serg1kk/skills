# TELEGRAM STYLE GUIDE TEMPLATE
## Guide for Writing Telegram Channel Posts

**Customize this template for your channel.** Replace examples with your own.

---

## MAIN PRINCIPLE

**Write as if a real person wrote this, excited about what they just did.**

Not AI text. Not marketing copy. A practitioner's living voice.

---

## POST STRUCTURE

### Typical structure:

```
**[Emoji] [Catchy title - specific, with a result]**

[1-2 sentences of context - what happened]

**What I did:**

[Specific steps with details]

**Result:**

[Outcome + insight]

[Optional: link or CTA]
```

### Structure rules:
1. **Title** - always specific, with numbers or results
2. **Context** - maximum 2-3 sentences
3. **Actions** - step by step, what exactly was done
4. **Result** - what was learned, what outcome
5. **CTA** - soft, not salesy

---

## TITLE FORMULAS

### Working formulas:

```
[What I did] in [time]. [Extra detail]
```
Examples:
- "Advent tree in 15 minutes. No code. On someone else's stack."
- "15 YouTube videos into a working guide in one day"

```
How I [action] using [tool]
```
Examples:
- "How I turned 15 YouTube videos into a working guide"

```
[Result] without [expected obstacle]
```
Examples:
- "Working feature in production. No programming skills."
- "MVP over a weekend. No dev team."

### DO NOT use:
- "Top 10 ways..."
- "Why you need..."
- "Secrets of successful..."
- Clickbait without substance

## TEXT RULES

### Length:
- **Short post**: 500-800 characters
- **Medium post**: 1000-1500 characters
- **Long post (case study)**: 1500-2500 characters
- **Maximum**: 3000 characters (rarely)

### Sentences:
- Short. Punchy. Specific.
- One idea = one sentence
- Break up long sentences

**Good:**
```
Got the archive. Opened Claude Code. Three prompts. Fifteen minutes.
```

**Bad:**
```
After I received the archive from the client, I opened Claude Code and in three prompts over fifteen minutes I did everything.
```

### Paragraphs:
- Short (2-4 sentences)
- Blank line between semantic blocks
- Use subheadings for structure

### Numbers and facts:
- Always specific: "3 hours", "9 screenshots", "15 videos"
- Tool names: "Gemini 2.5 Pro", "Claude Code", "Vercel"
- Technical details where appropriate

## FORMATTING IN TELEGRAM

### Markdown:
```markdown
**bold**          - for titles and key phrases
__italic__        - for quotes and highlights
`monospace`       - for code and technical terms
```

### Links:
**IMPORTANT: Always use bare URLs, NOT markdown links!**

Telegram does NOT render markdown links `[text](url)` - they display as raw text with brackets. Looks broken.

**Correct:**
```
Repo with agents (30k stars):
https://github.com/example/agents/

-> More on the blog:
https://yourdomain.com/blog/slug
```

**Incorrect:**
```
Repo: [github.com/example](https://github.com/example/agents/)
```

### Emoji:
- Moderate (1-3 per post)
- In title - 1 emoji
- For list structure - use relevant emoji
- Do NOT spam emoji in every sentence

### Dashes:
- **NEVER use em-dashes** - it's an AI generation marker
- Use only short dash (-) or restructure the sentence
- Instead of "Result - excellent" write "Result: excellent"

## POST TYPES

### 1. CASE STUDY / Practical Example
```
**[Emoji] [What I did in what time]**

[Context - what I wanted, what problem]

**What I did:**
[Step 1]
[Step 2]
[Step 3]

**Result:**
[Outcome]
[Insight/conclusion]

-> [Link to result]
```

### 2. TOOL / Discovery
```
**[What I found/tried]**

[Context - why I was looking]

**The trick:**
[What's special]

**How I used it:**
[Concrete example]

[Conclusion/recommendation]
```

### 3. INSIGHT / Observation
```
[Short observation]

[Context or example]

[Conclusion]
```
Length: 300-600 characters

### 4. ANNOUNCEMENT / Link
```
**[What I did]**

[1-2 sentences about what's inside]

-> [Link]
```
