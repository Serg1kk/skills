# Elite Agent Prompt Framework

## Назначение

Мета-промпт для создания высококачественных системных промптов AI-агентов. Используется когда нужно создать промпт для:
- AI-агентов с tools (n8n AI Agent, Claude с tools, OpenAI Assistants)
- AI-ассистентов без tools (chatbots, специализированные помощники)
- Любых LLM-based решений с системным промптом

## Когда использовать

Применяй этот фреймворк когда пользователь просит:
- Создать/написать промпт для агента
- Оптимизировать существующий системный промпт
- Сделать AI-ассистента для конкретной задачи
- Настроить AI Agent ноду в n8n
- Создать chatbot с определённой специализацией

## Как использовать (инструкция для LLM)

### Шаг 1: Загрузка фреймворка
Используй полный текст мета-промпта из `elite-agent-prompt.txt` (включая все XML-теги) как базу для создания промпта.

### Шаг 2: Сбор информации
Согласно фреймворку, ОБЯЗАТЕЛЬНО задай пользователю 3-5 критических вопросов:
- Конкретные цели и ожидаемые результаты
- Целевая аудитория и её уровень экспертизы
- Специфические ограничения или требования
- Требуемый уровень детализации
- Edge cases которые нужно обработать

Если информация уже есть в контексте (например, из n8n workflow JSON или документации), используй её.

### Шаг 3: Создание промпта
На основе ответов создай промпт со следующей структурой:
1. **Title/Description** — позиционирование агента как эксперта в домене
2. **Instructions** — чёткие пошаговые инструкции
3. **Chain of Thoughts** — логическая цепочка рассуждений
4. **What Not To Do** — запреты и anti-patterns
5. **Few-Shot Examples** — примеры желаемого и нежелаемого поведения

### Шаг 4: Создание обязательных артефактов

При создании промпта для ассистента ВСЕГДА создавай следующие файлы в папке проекта.

#### Правило именования файлов

**ОБЯЗАТЕЛЬНО** добавляй префикс с именем агента ко всем файлам:

```
{agent-name}-agent-prompts.txt
{agent-name}-agent-input-spec.md
{agent-name}-agent-json-schema.json
{agent-name}-agent-model-recommendations.md
{agent-name}-agent-tbd-items.md
```

**Примеры:**
- Для chapters агента: `chapters-agent-prompts.txt`
- Для summary агента: `summary-agent-json-schema.json`
- Для keywords агента: `keywords-agent-model-recommendations.md`

Это позволяет различать файлы разных агентов в одной папке или при переносе.

#### 1. `{agent}-agent-prompts.txt` — Промпты (User + System)

Файл содержит два раздела. **User message СВЕРХУ** (удобнее для n8n интерфейса):

```
================================================================================
USER MESSAGE
================================================================================

{{ $json.variableName }}


================================================================================
SYSTEM MESSAGE
================================================================================

<prompt>
...полный системный промпт в XML-структуре...
</prompt>
```

**Важно:**
- User message сверху — только переменная, никаких описаний
- System message снизу — вся логика, инструкции, примеры

#### 2. `{agent}-agent-input-spec.md` — Спецификация входных данных
Описание что приходит в user message:
- Имя переменной
- Формат данных (JSON, текст, SRT)
- Структура и примеры
- Ограничения (мин/макс длина, кодировка)
- Источник данных

#### 3. `{agent}-agent-tbd-items.md` — Вопросы для обсуждения
Документ с открытыми вопросами, которые требуют обсуждения с командой:
- Неопределённые требования
- Альтернативные подходы
- Edge cases требующие решения
- История принятых решений

#### 4. `{agent}-agent-model-recommendations.md` — Рекомендации по моделям
Документ с анализом и рекомендациями:
- Расчёт токенов для типичных входных данных
- Сравнительная таблица моделей (контекст, цена, качество)
- Расчёт стоимости на запрос
- Рекомендуемая температура для разных сценариев
- Итоговые рекомендации (MVP/Production/Premium)

#### 5. `{agent}-agent-json-schema.json` — JSON Schema
Формальная схема структуры ответа ассистента:
- Все обязательные и опциональные поля
- Типы данных и ограничения
- Коды статусов и ошибок
- Примеры успешных и ошибочных ответов

## Ключевые принципы фреймворка

1. **Язык коммуникации** — ВСЕГДА отвечать на языке пользователя
2. **Вопросы обязательны** — никогда не пропускать этап сбора информации
3. **Адаптация под модель** — для маленьких моделей проще язык, для больших — сложнее
4. **Chain of Thoughts** — всегда включать пошаговое рассуждение
5. **Negative examples** — обязательно показывать что НЕ делать
6. **Safety** — интегрировать этические ограничения

## Структура промпта (шаблон вывода)

```xml
<prompt>
<title># [AGENT ROLE NAME]</title>

<description>
YOU ARE [EXPERT POSITIONING]...
</description>

<instructions>
## INSTRUCTIONS
1. [Instruction 1]
2. [Instruction 2]
...
</instructions>

<chain_of_thoughts>
## CHAIN OF THOUGHTS
1. [Step 1]
   1.1. [Sub-step]
...
</chain_of_thoughts>

<restrictions>
## WHAT NOT TO DO
- NEVER [restriction 1]
- NEVER [restriction 2]
...
</restrictions>

<example>
## FEW-SHOT EXAMPLES

Desired Example:
[Input] → [Output]

Undesired Example:
[Input] → [Wrong Output]
</example>
</prompt>
```
